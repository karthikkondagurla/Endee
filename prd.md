**Version**: 1.0

## 1. Executive Summary

**Pigeon** is an AI-powered news aggregation platform built on top of **Endee**, a high-performance open-source vector database. The project demonstrates four core use cases of vector search in a real-world consumer application:

1. **Semantic Search** – find articles by meaning, not just keywords
2. **RAG (Retrieval-Augmented Generation)** – ground Groq LLM answers in real news
3. **Recommendations** – surface similar articles automatically
4. **Agentic AI Workflow** – autonomous topic monitoring + digest generation

Pigeon is deployed on **Vercel** (Next.js frontend + API routes) and **Render** (Endee vector DB as a persistent service).

---

## 2. Problem Statement

Traditional news apps have three core problems:

| Problem | Impact |
|---|---|
| Keyword-only search | Misses semantically relevant articles |
| No personalization | Users see noise, not signal |
| Raw article dumps | No synthesis or insight |

Endee solves all three by storing articles as **384-dimensional embeddings**, enabling millisecond nearest-neighbor retrieval that understands *meaning*.

## 4. Core Use Cases (Endee-Powered)

### 4.1 Semantic Search

**What it does:** Converts the user's search query into a vector embedding and retrieves the closest articles stored in Endee — even if they share no keywords.

**Example:**
- Query: *"chip shortage impact"*
- Finds: *"TSMC cuts semiconductor output forecast"*, *"Auto industry warns on component supply"*
- Would **not** be found with keyword search

**Implementation:**
```
User query → getEmbedding(query) → 384-dim vector
               ↓
Endee index.search({ vector, topK: 20, spaceType: 'cosine' })
               ↓
Ranked results with similarity scores
```

**Endee config used:**
- Index: `pigeon_articles`
- Dimension: 384 (all-MiniLM-L6-v2)
- Space type: cosine similarity
- Precision: INT8 (quantized for speed + low RAM)
- Algorithm: HNSW (sub-millisecond at scale)

---

### 4.2 RAG — Retrieval-Augmented Generation

**What it does:** When a user asks a question like *"What happened with OpenAI this week?"*, Pigeon:
1. Embeds the question
2. Retrieves the top-5 relevant articles from Endee
3. Injects them as context into a Groq (Llama 3.1 8B) prompt
4. Returns a grounded, cited answer

**Why RAG matters:** The LLM cannot hallucinate — every claim is sourced from real fetched articles.

**Prompt template:**
```
You are a news analyst. Using ONLY the following articles, answer the question.
Cite sources by [1], [2], etc.

Articles:
[1] {article1.title}: {article1.content}
[2] {article2.title}: {article2.content}
...

Question: {userQuestion}

Answer (cite sources):
```

**API route:** `POST /api/rag`  
**Flow:** query → Endee search → top-k articles → Groq → streamed answer

---

### 4.3 Article Recommendations

**What it does:** When a user opens any article, Pigeon finds the 5 most semantically similar articles already in their feed and shows them as "You might also like."

**Implementation:**
```
Article opened → article.embedding (pre-stored in Endee)
    ↓
index.search({ vector: article.embedding, topK: 6 })
    ↓
Remove self (score = 1.0) → return top 5
```

**Deduplication:** Articles with cosine similarity > 0.85 are flagged as duplicates during ingestion and not added to Endee — preventing redundant recommended reads.

**UI:** "Similar Articles" panel at the bottom of the reader modal, each card showing a similarity score badge (e.g. "94% match").

---

### 4.4 Agentic AI Workflow — Daily Digest Agent

**What it does:** A background agent runs every 24 hours and autonomously:
1. Fetches latest RSS articles for all user topics
2. Embeds and upserts them into Endee
3. Searches Endee for the top-10 most relevant articles per topic
4. Generates a structured daily digest via Groq
5. Caches the digest in localStorage for instant access

**Agent loop:**
```
[TRIGGER] Scheduled cron (Vercel Cron Job, 6AM daily)
    ↓
[FETCH] fetchRSS(allTopics) → raw articles
    ↓
[EMBED] getEmbedding(title + snippet) for each article
    ↓
[UPSERT] Endee index.upsert(vectors + metadata)
    ↓
[RETRIEVE] Endee search per topic → top-10 articles
    ↓
[GENERATE] Groq: "Summarize today's top stories in {topic}"
    ↓
[CACHE] Store digest → localStorage / API response
    ↓
[DISPLAY] "Today's Briefing" card on dashboard
```

**This is agentic because:** The system autonomously decides what to fetch, embed, retrieve, and generate — without any user interaction — following a goal-directed loop.

### 5.2 Interaction Details

#### Search Flow
1. User types in search bar (debounced 300ms)
2. Query sent to `POST /api/endee/search`
3. Server generates embedding via `all-MiniLM-L6-v2`
4. Endee returns ranked results with scores
5. UI renders results with similarity badge

#### Article Ingestion Flow
1. `fetchRSS(topics)` pulls Google News RSS
2. Each article: `title + " " + contentSnippet` → embedding
3. Upsert to Endee with payload: `{ title, link, topic, pubDate, sourceName }`
4. Dedup check: if cosine similarity > 0.85 with existing → skip

## 9. Data Model

### Endee Vector Payload Schema

Each upserted article vector stores metadata in the Endee payload:

```typescript
interface ArticlePayload {
  id: string;          // SHA-256 hash of URL (dedup key)
  title: string;
  link: string;
  topic: string;       // User topic that triggered fetch
  sourceName: string;  // News outlet name
  pubDate: string;     // ISO 8601
  contentSnippet: string;
  imageUrl?: string;
  saved?: boolean;     // Bookmark state
}
```

### Embedding Generation

```typescript
// Input: article title + snippet concatenated
const text = `${article.title} ${article.contentSnippet}`;

// Model: all-MiniLM-L6-v2 (23MB, cached after first load)
// Output: 384-dimensional float32 vector, L2-normalized
const embedding = await getEmbedding(text);
```

## 11. Deployment

### 11.1 Endee on Render

Endee runs as a long-lived **Web Service** on Render to persist the vector index between requests.

**Steps:**
1. Create a new **Web Service** on [render.com](https://render.com)
2. Connect the Endee GitHub repository (or use Docker)
3. Set build command:
   ```bash
   cmake -B build -DCMAKE_BUILD_TYPE=Release && cmake --build build -j4
   ```
4. Set start command:
   ```bash
   ./build/endee --port 8080 --data-dir /data
   ```
5. Add a **Persistent Disk** (at least 1GB) mounted at `/data` to survive restarts
6. Set environment:
   ```
   PORT=8080
   ```
7. Note the public URL: `https://endee-pigeon.onrender.com`

**Render Service Settings:**
| Setting | Value |
|---|---|
| Environment | Docker / Native |
| Plan | Starter ($7/mo) — required for persistent disk |
| Region | Oregon (US West) |
| Health check path | `GET /api/v1/collections` |
| Auto-deploy | Enabled |

---

### 11.2 Next.js on Vercel

**Steps:**
1. Push code to GitHub
2. Import repository at [vercel.com/new](https://vercel.com/new)
3. Framework preset: **Next.js** (auto-detected)
4. Add environment variables in Vercel dashboard:
   ```
   GROQ_API_KEY         → your Groq key
   ENDEE_URL            → https://endee-pigeon.onrender.com/api/v1
   NEXT_PUBLIC_APP_URL  → https://pigeon.vercel.app
   ```
5. Deploy — Vercel handles builds automatically on every push to `main`

**Vercel Configuration (`vercel.json`):**
```json
{
  "functions": {
    "src/app/api/news/route.ts": {
      "maxDuration": 60
    },
    "src/app/api/endee/upsert/route.ts": {
      "maxDuration": 60
    },
    "src/app/api/rag/route.ts": {
      "maxDuration": 30
    }
  },
  "crons": [
    {
      "path": "/api/cron/digest",
      "schedule": "0 6 * * *"
    }
  ]
}
```

> **Note:** The embedding pipeline (`@xenova/transformers` + ~23MB model download) runs on Vercel serverless. Set `maxDuration: 60` for routes that call `getEmbedding()` on cold start.

## 14. Constraints & Assumptions

### Technical Constraints
| Constraint | Detail |
|---|---|
| Endee requires persistent process | Cannot run as serverless — must use Render with disk |
| Vercel function cold start | First `getEmbedding()` call takes ~3s (model download) |
| Groq rate limits | 60 req/min on free tier |
| Vercel function timeout | 60s max (sufficient for embedding + Groq) |
| Endee INT8 quantization | Slight accuracy tradeoff for 4× memory reduction |

### Assumptions
- English-language news (initial launch)
- Users provide their own Groq API key (BYOK model)
- Single-user deployment initially (no per-user vector isolation)
- Modern browsers (Chrome 90+, Firefox 88+, Safari 14+)

## 15. Glossary

| Term | Definition |
|---|---|
| HNSW | Hierarchical Navigable Small World — graph-based approximate nearest neighbor algorithm used by Endee |
| RAG | Retrieval-Augmented Generation — grounding LLM responses in retrieved documents |
| Embedding | 384-dimensional float vector representing the semantic meaning of text |
| Cosine Similarity | Angle-based similarity between two vectors; 1.0 = identical, 0.0 = unrelated |
| INT8 Quantization | Compressing float32 vectors to 8-bit integers — 4× smaller, ~1% accuracy loss |
| BYOK | Bring Your Own Key — users supply their Groq API key |
| Agentic Workflow | A multi-step autonomous AI loop that executes without per-step human input |
| all-MiniLM-L6-v2 | 23MB sentence embedding model producing 384-dim vectors, used for all embeddings |
